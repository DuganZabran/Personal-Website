Creating My Personal Website: A Journey in HTML, CSS, Bootstrap, and AWS


Tools and Technology:
  • HTML and CSS for front-end development
  • Bootstrap for responsive design and UI components
  • Amazon S3, Route 53, ACM, and CloudFront for web hosting and delivery
  • GitHub for version control and collaboration
  • GitHub Actions for continuous integration and deployment

You can check out the code for my website on my GitHub repository: https://github.com/DuganZabran/Personal-Website

Building a website is an exciting challenge that requires a mix of creativity, technical skills, and persistence. In my case, I wanted to create a personal website that showcased my skills, interests, and projects in a simple and elegant way. After researching various tools and frameworks, I settled on using HTML, CSS, Bootstrap, Amazon Web Services (AWS), and GitHub Actions to build and deploy my site.

Front-End Development with HTML, CSS, and Bootstrap

To create the look and feel of my website, I used HTML and CSS to write the markup and styles for each page. I started by sketching out a wireframe of the site's layout, which helped me to plan the structure and content of each section. Then, I coded each page using semantic HTML tags, such as <header>, <main>, <nav>, <section>, and <footer>, to provide meaningful structure to the content. I also used CSS to style the typography, colors, spacing, and layout of the site, following the principles of responsive design to ensure that it looked good on different screen sizes and devices.

To speed up my development process and improve the consistency of my design, I also used Bootstrap, a popular front-end framework that provides pre-built HTML and CSS components, such as buttons, forms, cards, modals, and grids. By using Bootstrap's classes and styles, I was able to create a professional-looking site without spending too much time styling or reinventing the wheel.

Back-End Infrastructure with AWS

Once I had the front end of my site working locally, I needed to deploy it to the web so that others could access it. For this, I chose to use AWS, a cloud computing platform that offers a range of services for hosting, managing and scaling web applications.

To store my website's files and assets, I used Amazon S3, a simple storage service that allows you to upload and download objects (files) via HTTP or HTTPS. I created a new S3 bucket and configured it to host a static website, which involved setting the bucket policy, creating an index document, and enabling website hosting. Then, I uploaded my HTML, CSS, and image files to the bucket, and voila! My site was live on the internet.

To add a custom domain name to my site, I used Amazon Route 53, a domain name system (DNS) service that allows you to register, manage, and route domain names to your resources. I purchased a domain name from a registrar and then created a new hosted zone in Route 53, which involved adding the domain name and specifying the name servers. Then, I created a new record set that mapped my domain name to my S3 bucket endpoint and waited for the DNS propagation to complete.

o secure my website's traffic with SSL/TLS encryption, I used Amazon Certificate Manager (ACM), a service that provides free SSL/TLS certificates for your domains and subdomains. I requested a new certificate for my domain name in ACM and validated it by adding a CNAME record to my Route 53 hosted zone. Once the certificate was issued, I enabled HTTPS on my S3 bucket and CloudFront distribution, which acts as a content delivery network (CDN) that caches and serves my website's files from edge locations around the world. This not only improves the site's performance and reliability but also ensures that visitors can access it securely via HTTPS.

CI/CD with GitHub Actions

Finally, to automate the process of updating my S3 bucket whenever I make changes to the code, I used GitHub Actions, a feature of GitHub that allows you to run custom workflows in response to events, such as a new commit or a pull request. I created a new workflow that listens to changes in my main branch and authenticates with AWS using an IAM user that I created specifically for this purpose. This workflow then uses the AWS CLI to sync the generated files with my S3 bucket. This ensures that my site is always up-to-date and consistent with my latest changes, without requiring me to manually upload the files each time.

Conclusion

Creating my personal website was a challenging and rewarding experience that taught me a lot about web development, AWS, and GitHub Actions. By using HTML, CSS, Bootstrap, AWS, and GitHub Actions, I was able to build and deploy a simple and elegant site that showcases my skills and interests. I hope that this post has been helpful and informative for anyone who is interested in creating their own website or learning more about web development. Feel free to check out the code for my site on GitHub and let me know if you have any feedback or questions!
